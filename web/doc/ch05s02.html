5.2. Output Hierarchy

disktype yields a JSON object for each analyzed source. This object takes the hierarchical structure of the source into account and represents the content accordingly. That says, partitions that are found next to each other on a disk will also be next to each other in some content list of the output. However, a file system found inside a partition will be in a sub-content list assigned to this partition. That way the structure is maintained.

The JSON object consists of 4 key-value pairs with the keys file kind, path, and size with strings as values, as well as content which is an array.

Example: A result where no structure at all was detected, may look like this: 

                {
                   "file kind": "Regular file",
                   "path": "/some-path/my-image.image",
                   "size": "1024",
                   "content": []
                }

file kind holds either Regular file for any ordinary file, Block device for any device that conveys data in blocks through some buffer, Character device for any device that conveys data character by character or Unknown kind if it isn't recognized.

path contains the path to the given source.

size contains the total number of bytes of the given source.

content contains an array of so called content objects that each represent a structure found in the source.
